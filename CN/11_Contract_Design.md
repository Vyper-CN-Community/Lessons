# 设计你的第一个 Vyper 智能合约：规划与需求

在开始编写复杂代码之前，特别是对于像智能合约这样的不可变系统，仔细的规划是必不可少的。准确理解你的合约需要实现什么目标是关键的第一步。本课概述了使用 Vyper 编程语言设计一个简单的“喜爱事物列表”智能合约的初始设计阶段。

我们将从 Vyper 集成开发环境 (IDE) 中开始。当开始一个新的 Vyper 合约文件（通常以 `.vy` 为扩展名）时，你经常会看到一些初始行：

```vyper
# pragma version 0.4.0
# @license MIT
```

`# pragma version` 指令指定了合约所适用的 Vyper 编译器版本。这有助于确保兼容性，并防止未来版本更改编译器行为时出现问题。`# @license` 行使用 SPDX 许可证标识符（在本例中为 MIT）声明代码发布的开源许可证。Vyper 中的注释由 `#` 符号表示。

有了基本的文件结构，下一个至关重要的步骤是定义合约的需求。我们究竟希望这个智能合约做什么？对于我们的例子，我们的目标是创建一个具有两个主要功能的“喜爱事物列表”：

1.  存储一个通用的喜爱数字列表。
2.  存储特定的人，并将每个人与他们各自的喜爱数字关联起来。

为了在开发过程中始终关注这些需求，一个好的做法是**直接使用注释在代码文件中列出大纲**。这充当了嵌入在源代码本身中的初始设计文档：

```vyper
# pragma version 0.4.0
# @license MIT

# Favorite Things List
# 1. Store a number
# 2. Store a person's name and favorite number
```

在这里，我们添加了注释，概述了核心目的（喜爱事物列表）及其两个特定的数据存储需求。这个简单的规范清楚地陈述了我们的目标：我们需要一种机制来保存数字集合，以及另一种机制来映射标识符（代表人）到特定数字。
